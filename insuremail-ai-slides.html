02
The Problem

Insurance inbox = operational liability at scale

Volume & Speed

Large insurers receive 10,000–50,000 emails/day. Human agents triage each in 2–5 minutes. At scale, this is a full operations centre just to read mail.

Routing Accuracy

Manual routing averages ~85% accuracy. 15% misrouted emails create re-work, SLA breaches, and frustrated customers who have to repeat themselves.

PII Risk

Emails contain policy numbers, IRNs, dates of birth, bank details. Unprocessed inboxes sitting in a shared mailbox are a GDPR liability waiting to happen.

Peak-hour Bottleneck

Monday 9–11am: claims surge after weekends. Human teams hit capacity. Response SLAs breach. Complaint volumes spike. No elastic solution exists in a manual process.

Why Hybrid?

Test Strategy — How We Measured

3-Layer Test Pyramid
Layer 1 · Unit Tests
45 tests · 78%
  • pytest · all 10 Lambda functions tested in isolation
  • moto v5 @mock_aws — DynamoDB · S3 · Step Functions without real AWS
  • Env vars in conftest.py before module import · module isolation per test
Layer 2 · Evaluation Scripts
100-record sample
  • Local eval: intent accuracy · routing precision · confidence calibration · entity extraction
  • LLM-as-judge: Bedrock Haiku scores each output on 8 quality dimensions
  • TruLens RAG Triad: faithfulness · answer relevance · context relevance
Layer 3 · Live Pipeline Assessment
20-email batch
  • StepFunctionAssignment.py: fires real emails through Step Functions end-to-end
  • Measures: pipeline success rate · routing accuracy · CRM hit rate · latency
  • Results uploaded to S3 → fetched by React dashboard for live display
What Each Layer Catches
Unit — logic bugs in isolation
Lambda handler errors · DynamoDB schema mismatches · missing env vars · wrong model ID · Decimal type handling
Eval Scripts — AI quality issues
Intent misclassification · overconfident predictions · hallucinated responses · poor RAG retrieval · routing errors by class
Live Pipeline — integration failures
Step Functions state transitions · Lambda timeouts · CRM lookup failures · end-to-end latency · real Bedrock throttling
